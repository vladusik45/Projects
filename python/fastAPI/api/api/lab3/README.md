Ссылка на репозиторий одногруппника: https://gitverse.ru/leof/library-rest-api/content/master

В API доступно:
/api/v1/books — CRUD для книг
/api/v1/users — управление пользователями
/api/v1/loans — управление выдачей книг
Документация доступна через Swagger / OpenAPI.

Структура:
client.py — основной класс клиента для взаимодействия с API
models.py — Pydantic-модели для структурирования ответов
examples.py — примеры использования клиента

Клиент поддерживает:
аутентификацию через JWT / API-ключ
базовые CRUD-операции для ресурсов books, users, loans
пагинацию и лимиты запросов 
обработку ошибок 
идемпотентную операцию 

Проблемы:
Не все примеры POST-запросов содержат полное тело.
Не все POST-эндпоинты поддерживают заголовок Idempotency-Key, пришлось реализовать на стороне клиента повторную проверку.
Документация не всегда указывает максимальные значения limit.

Отзыв:
API структурировано, REST-методы соответствуют CRUD, легко интегрировать, документация в основном нормальная.
